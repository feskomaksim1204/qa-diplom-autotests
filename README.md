# QA Диплом: от модели до тестового фреймворка

## Описание проекта

Это дипломный проект, который я создавал, чтобы **показать свой прогресс в автоматизации тестирования**: от простой Java-модели до полноценного тестового фреймворка.

**Какие проблемы решает проект?**  
В реальной разработке часто сначала пишут код, а потом забывают его тестировать. Я решил показать обратный путь: сначала написать бизнес-логику (модель бургерной), затем покрыть её API-тестами (REST Assured) и в итоге построить UI + API фреймворк с Selenium, Page Object и Allure. Компании тратят часы на ручное регрессионное тестирование. Мой проект демонстрирует, что я умею создавать автотесты, которые экономят время команды и находят критические баги до релиза.

**Почему я его создавал?**  
Чтобы доказать (в первую очередь себе и будущему работодателю), что я могу не только выполнять ручные проверки, но и проектировать тестовую архитектуру, писать поддерживаемые автотесты и настраивать отчёты.

**Стек технологий:**  
- Java 11  
- JUnit 4  
- Selenium WebDriver  
- REST Assured  
- Maven  
- Allure  
- Page Object Pattern

## Системные требования и запуск

**Требования:**  
- Java 11 или выше  
- Maven 3.9+  
- Браузеры: Chrome (последняя версия) или Яндекс Браузер

**Запуск API-тестов заказов (папка `02-api-order-tests`):**

    cd 02-api-order-tests
    mvn clean test
    mvn allure:report

**Запуск UI + API автотестов (папка `03-ui-stellar-burgers`):**

    cd 03-ui-stellar-burgers
    mvn clean test
    mvn clean test -Dbrowser=yandex
    mvn allure:report

## Вывод

Проект демонстрирует мои навыки в автоматизации тестирования на Java: от написания простой модели до построения фреймворка с поддержкой кроссбраузерности и генерации отчётов. Это не просто «учебный проект», а доказательство моей готовности решать реальные задачи в коммерческой разработке.
